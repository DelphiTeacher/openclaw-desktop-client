# 第一步是自检配置是否OK
先检测openclaw是不是配置好了，如果在用户目录中存在.openclaw文件夹，则说明openclaw配置好了。
如果没有配置好，则跳转到配置页面配置大模型即可，配置好后调用openclaw onboard命令来保存配置到openclaw。


# 第二步是启动openclaw gateway
有两种启动服务的方式，一种是直接运行命令，另一种是安装成服务。
安装成服务的方式，但是这种情况的话，是运行在SYSTEM用户下，openclaw在读取配置的时候会读取不到当前用户的配置。
会报错：2026-03-21T17:37:17.785+08:00 Missing config. Run `openclaw setup` or set gateway.mode=local (or pass --allow-unconfigured).
所以还是用运行命令的方式启动服务，保持在同一用户下。



# 第二步是检测openclaw gateway是否启动
先检测openclaw gateway是不是启动了，如果没有启动，则要先启动它。
如何判断gateway是不是已经启动了？去连接它的服务端口吗？
因为websocket用到了三方控件，所以暂时可以使用遍历进程的方式来判断，这样不需要引用三方控件。



# 第三步使用webbrowser浏览openclaw页面
从openclaw.json中获取网关的token，拼在网页后面就可以在webbrowser中访问openclaw页面了。
然后从.openclaw/openclaw.json中获取gateway的地址和端口号和token，
再启动。
